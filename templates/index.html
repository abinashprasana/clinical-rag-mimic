Clinical RAG

Welcome to my final NLP project exploring how we can use large language models to answer medical questions. This system reads real hospital discharge notes from the MIMIC database and only uses that factual information to formulate an answer. I built it so it declines to guess if the answer is missing, which is a really important safety feature in healthcare.

Section 1: About the Dataset

For this project I used the MIMIC discharge notes dataset which contains thousands of records from real hospital visits. I started by looking at how long these notes usually are to figure out the best way to process them. I also filtered the huge dataset down into three main medical conditions to focus my analysis.

Section 2: Data Exploration

I wanted to see how the doctors actually structured their notes so I tracked how often standard medical headers appeared. It was really interesting to see which sections were the most common. I also looked at the most frequent words before and after I cleaned up the text to make sure my preprocessing pipeline was removing all the unhelpful noise and keeping the important medical terms.

Section 3: Word Clouds

To get a better visual sense of the vocabulary I created these word clouds. The first one shows the most common words across all the notes in the whole dataset. The second one focuses just on the notes from patients with diabetes which is the subset I chose for the final question answering system.

Section 4: Pipeline Architecture

The system works by breaking down long medical notes into smaller overlapping chunks and converting each chunk into a mathematical vector that represents its meaning. When a user asks a question the system converts the question into a vector too and searches for the most similar chunks from the database using a fast search index. It then hands those specific chunks over to a language model and tells it to generate a clear answer using only the provided text so it does not make up false medical advice.

Section 5: Evaluation Results

I tested the system on ten standard medical questions to see how well it could find the right information. The accuracy chart shows whether the system successfully included the expected key medical terms in its answer. Finding the right term around sixty percent of the time is actually quite good for this model size because sometimes it uses a valid alternative word that my simple keyword checker missed. The latency chart shows how long each question took to process which changes depending on how many chunks the model had to read through.
